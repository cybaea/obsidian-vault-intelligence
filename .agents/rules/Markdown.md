# Markdown Formatting Rules

When editing or generating Markdown files (`*.md`), strictly adhere to the following constraints:

1. **Language & Spelling:** Use British English with Oxford dictionary rules (e.g., `en-GB-oxendict`).
2. **Flavor:** Use GitHub Flavored Markdown (GFM) as the baseline. If writing specifically for Obsidian notes, use Obsidian-flavored markdown (wikilinks, callouts).
3. **Typography & Styling:**
   - Prefer sentence case for all headers.
   - Use bold text sparingly and only for emphasis.
   - **Never** use bold formatting inside markdown headers.
   - Prefer 'and' over '&' in running text.
4. **Emojis:** Use emojis extremely sparingly. Never use emojis for decoration in headers or bullet points.
5. **Code Blocks:** Always specify the language for syntax highlighting.
